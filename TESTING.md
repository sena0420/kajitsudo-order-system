# プロジェクトテスト手順ガイド

## はじめに

このガイドでは、スーパーバイヤー向け発注システムをテストする方法を詳しく説明します。

## 前提条件

- Node.js (v16以上) がインストールされている
- npm (v8以上) がインストールされている
- モダンなブラウザ (Chrome, Firefox, Safari, Edge)

## 1. 環境のセットアップ

### Step 1: 依存関係のインストール

```bash
# プロジェクトディレクトリに移動
cd order-system

# 依存関係をインストール
npm install
```

### Step 2: 環境変数の設定（オプション）

Firebase を使用する場合は、`.env` ファイルを作成してください：

```bash
# .env.example をコピー
cp .env.example .env
```

`.env` ファイルを編集してFirebase設定を追加（現在はモックデータで動作するため省略可能）：

```
REACT_APP_FIREBASE_API_KEY=your-api-key
REACT_APP_FIREBASE_AUTH_DOMAIN=your-project.firebaseapp.com
REACT_APP_FIREBASE_PROJECT_ID=your-project-id
REACT_APP_FIREBASE_STORAGE_BUCKET=your-project.appspot.com
REACT_APP_FIREBASE_MESSAGING_SENDER_ID=123456789
REACT_APP_FIREBASE_APP_ID=1:123456789:web:abcdef
```

## 2. 開発サーバーの起動

### 方法1: npmスクリプトを使用

```bash
npm start
```

### 方法2: npxを直接使用（推奨）

```bash
npx react-scripts start
```

### 方法3: Windowsで問題が発生する場合

```bash
# npmキャッシュをクリア
npm cache clean --force

# node_modules を削除して再インストール
rmdir /s node_modules
del package-lock.json
npm install

# 再度起動を試す
npm start
```

## 3. ブラウザでの動作確認

開発サーバーが起動すると、通常は自動的にブラウザが開きます。
手動で開く場合は以下のURLにアクセス：

```
http://localhost:3000
```

## 4. 機能テスト手順

### 4.1 ログイン機能のテスト

1. **ログイン画面の表示確認**
   - ブラウザでアプリにアクセス
   - ログインフォームが表示されることを確認
   - レスポンシブデザインの動作確認（画面サイズを変更）

2. **ログイン処理のテスト**
   - 顧客ID: `CUST001`（任意の値でOK）
   - パスワード: `password`（任意の値でOK） 
   - 「ログイン」ボタンをクリック
   - 発注画面に遷移することを確認

### 4.2 商品発注機能のテスト

1. **商品一覧の表示確認**
   - 商品カードが表示されることを確認
   - 「よく発注」チップが表示される商品があることを確認
   - 商品情報（名前、規格、産地、価格、納期）が正しく表示されることを確認

2. **数量調整機能のテスト**
   - 「+」「-」ボタンで数量を調整
   - 数量が正しく表示されることを確認
   - 数量が0以下にならないことを確認

3. **カート機能のテスト**
   - 商品を選択すると画面下部に発注ボタンが表示されることを確認
   - 合計金額が正しく計算されることを確認
   - 選択した商品数が表示されることを確認

4. **発注確認画面のテスト**
   - 「発注」ボタンをクリック
   - 発注確認ダイアログが表示されることを確認
   - 選択した商品と数量、金額が正しく表示されることを確認
   - 「発注確定」をクリック
   - 成功メッセージが表示されることを確認
   - カートがクリアされることを確認

### 4.3 発注履歴機能のテスト

1. **履歴画面への遷移**
   - ナビゲーションバーの「履歴」ボタンをクリック
   - 発注履歴画面に遷移することを確認

2. **履歴一覧の表示確認**
   - 過去の発注履歴が一覧で表示されることを確認
   - 発注日、納期、合計金額、ステータスが表示されることを確認
   - ステータス別にアイコンと色分けがされていることを確認

3. **履歴詳細の表示確認**
   - 履歴項目をクリックして展開
   - 発注した商品の詳細情報が表示されることを確認
   - 作業コード、商品名、規格、産地、数量、単価、小計が表示されることを確認

### 4.4 レスポンシブデザインのテスト

1. **デスクトップ表示**
   - 画面を最大化してレイアウトが適切に表示されることを確認

2. **タブレット表示**
   - ブラウザの開発者ツールでタブレットサイズ（768px）にリサイズ
   - レイアウトが適切に調整されることを確認

3. **スマートフォン表示**
   - ブラウザの開発者ツールでスマートフォンサイズ（375px）にリサイズ
   - モバイルレイアウトが適切に表示されることを確認
   - 発注ボタンが画面幅いっぱいに表示されることを確認
   - ナビゲーションボタンが小さくまとまって表示されることを確認

### 4.5 UI/UXの確認

1. **視覚的な確認**
   - カラーテーマが適切に適用されていることを確認
   - アイコンが正しく表示されることを確認
   - フォントサイズが読みやすいサイズになっていることを確認

2. **操作性の確認**
   - ボタンが十分な大きさでタッチしやすいことを確認
   - ローディング状態が適切に表示されることを確認
   - エラーメッセージが適切に表示されることを確認

## 5. エラーが発生した場合のトラブルシューティング

### 5.1 よくあるエラーと解決方法

#### npm start が動作しない

```bash
# 解決方法1: キャッシュクリア
npm cache clean --force

# 解決方法2: node_modules再インストール
rm -rf node_modules package-lock.json
npm install

# 解決方法3: npxを使用
npx react-scripts start
```

#### ブラウザでページが表示されない

1. ポート3000が他のアプリケーションで使用されていないか確認
2. ファイアウォールでポート3000がブロックされていないか確認
3. 手動でhttp://localhost:3000にアクセス

#### Firebaseエラーが発生する

現在のバージョンはモックデータを使用しているため、Firebaseのエラーは無視して構いません。

### 5.2 ログの確認方法

```bash
# 開発サーバーのログを確認
# コンソールに表示されるメッセージを確認

# ブラウザの開発者ツールを開く
# F12キー または 右クリック → 「検証」
# Consoleタブでエラーメッセージを確認
```

1. ブラウザの開発者ツールでエラーを確認
2. コンソールログを確認
3. READMEファイルの「サポート」セクションを参照
4. GitHubのIssuesで報告
